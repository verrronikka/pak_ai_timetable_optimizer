# UI

## Открытие экрана

1. Открыть `frontend/index.html`.
2. Убедиться, что загрузился заголовок экрана и таблица расписания.

## Сценарии demo query

1. `?demo=ready` — есть строки расписания, статус "Готово", баннер ошибки скрыт.
2. `?demo=loading` — таблица показывает skeleton, статус "Генерация идёт".
3. `?demo=empty` — таблица пустая, отображается empty-state.
4. `?demo=error` — показывается общий баннер ошибки.
5. `?demo=idle` — статус "Нет активной генерации", пустая таблица.

## UI-действия генерации

1. Кнопка "Сгенерировать" переводит экран в loading, затем в выбранный режим результата.
2. Кнопка "Обновить" перерисовывает текущий сценарий без поломки состояния.
3. Кнопка "Сбросить" переводит экран в `idle`.

## Базовая визуальная проверка

1. Таблица остается читаемой на ширине ~1280px и ~375px.
2. Статус-плашка меняет состояние (`idle/pending/running/completed/failed`).
3. Ошибки JavaScript в консоли отсутствуют.
